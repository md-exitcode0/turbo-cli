Sparse V 50-Chunk 32K PPL Validation (wikitext-103)
====================================================

Date: 2026-03-27
Hardware: Apple M5 Max 128GB
Model: Qwen3.5-35B-A3B Q8_0 (MoE)
Branch: feature/turboquant-kv-cache @ a52586e (post-PR#4-revert)
Corpus: wikitext-103-raw-v1 train split (516MB, ~135M tokens)
Context: 32768 tokens
Chunks: 50
Sparse V threshold: τ = 1e-6

Methodology:
  - Three sequential runs on clean GPU (no competing processes)
  - q8_0 baseline first to anchor the experiment
  - turbo3 OFF second to isolate TurboQuant compression overhead
  - turbo3 ON third to isolate sparse V effect
  - All runs use identical corpus, context length, chunk count, and model

Commands:
  WIKI103=wikitext-103-raw/wiki.train.raw

  # 1. q8_0 baseline
  llama-perplexity -m $MODEL -f $WIKI103 -c 32768 \
    -ctk q8_0 -ctv q8_0 -fa on --chunks 50 -ngl 99

  # 2. turbo3 sparse V OFF
  TURBO_SPARSE_V=0 llama-perplexity -m $MODEL -f $WIKI103 -c 32768 \
    -ctk turbo3 -ctv turbo3 -fa on --chunks 50 -ngl 99

  # 3. turbo3 sparse V ON
  llama-perplexity -m $MODEL -f $WIKI103 -c 32768 \
    -ctk turbo3 -ctv turbo3 -fa on --chunks 50 -ngl 99

Results:
  1. q8_0 (standard KV cache, no TurboQuant, no sparse V):
     PPL = 7.0638 +/- 0.02097

  2. TurboQuant turbo3 WITHOUT sparse V (compression only, no skip):
     PPL = 7.1796 +/- 0.02135

  3. TurboQuant turbo3 WITH sparse V (compression + attention-gated V skip):
     PPL = 7.1796 +/- 0.02135

Analysis:
  - q8_0 baseline: 7.0638 (sane, tight CI)
  - TurboQuant compression overhead (run 2 vs run 1): +1.64% vs q8_0
  - Sparse V effect (run 3 vs run 2): 0.0000 — enabling sparse V does not
    change PPL compared to TurboQuant without sparse V
  - Confidence interval ±0.021 — sufficient statistical power to detect
    a difference of ~0.04 PPL (0.6%). No difference observed.
  - At 32K context, estimated ~90% of V positions are skipped by sparse V

Note on earlier 2-chunk / 5-chunk results:
  Prior runs with wikitext-2 (1.3MB corpus) were limited to 2-5 chunks at 32K.
  This run uses wikitext-103 (516MB) for 50 chunks, providing 10-25x more
  statistical power. Results are consistent: ON/OFF delta remains 0.0000.
